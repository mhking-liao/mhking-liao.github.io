人生的幸福，一半要争，一半要随。争，不是与他人，而是与困苦。没有唾手可得的幸福，发愤图强，主动争取才能一步步接近幸福。随，不是随波逐流，而是知止而后安。能力与条件的限制，很多人事只能随遇而安，随缘而止。争，人生少遗憾；随，知足者常乐。最怕该争时不争，该止时不止，总在纠结中痛苦着。

一月 1, 1
Git上传、覆盖、删除 然后通过以下指令初始化，把这个项目所在目录变成Git可以管理的仓库。 git init 成功后，复制仓库的ssh地址，并输入以下指令： git remote add origin 仓库ssh地址 也可以用以下指令删除关联： git remote rm origin 也可以通过以下指令更换仓库： git remote set-url origin 仓库ssh地址 更换后，可以用查询查看是否更换成功。 添加上传 将全部内容添加，先输入以下指令： git add . 如果是指定目录，就改为以下指令： git add 目录 如果是指定文件，就改为以下指令： git add 文件.后缀 之后，再输入以下指令： git commit -m "把文件提交，这里是注释" 到这里，你的添加并没有上传到仓库，只是添加到了暂存区，上传到仓库还需要输入以下指令： git push -u origin main 最后，还会有个提示，输入 yes 即可。 删除 通过以下指令删除指定文件： git em 文件.后缀 通过以下指令可以删除指定文件夹： git em -r 文件夹 最后，必要通过以下指令提交修改： git commit -m "注释" 查询 查看本地分支： git branch 查看关联仓库地址： git remote -v 待补充。

hugo安装 一、安装Git sudo apt install git 配置ssh二、安装go mkdir ~/home_opt #home下软件安装包位置的home_opt，可设置自己的位置 cd ~/home_opt wget -c https://go.dev/dl/go1.17.3.linux-amd64.tar.gz sudo tar -xz go1.17.3.linux-amd64.tar.gz mv -f go go_install #重命名 sudo mv -f go_install /usr/local #移动到安装区 cd /usr/local sudo chmod -R 0777 go_install 添加到$PATH环境变量GOROOT是系统上安装Go软件包的位置GOPATH是工作目录的位置# src：go在包安装、文件编译时自动寻找区，bin：go执行命令区 mkdir ~/home_opt/go_work ~/home_opt/go_work/src ~/home_opt/go_work/bin #go工作区 sudo chmod -R 0777 go_work 修改环境变量，我的shell改为zsh了。sudo gedit ~/.zshrc #内容： export GOROOT=/usr/local/go_install export GOPATH=~/home_opt/go_work export PATH=$GOPATH/bin:$GOROOT/bin:$PATH source ~/.zshrc #环境变量生效 export #输出 默认shell的修改使用下面的：echo $shell #查看默认shell sudo gedit ~/.bashrc source ~/.
